# ドリルスルー（金額セルクリック → 明細科目×部署の内訳表示）

- 優先度: Must
- ステータス: 未着手
- 対象: 解析画面の集計テーブル
- PRD: §7.7 ドリルダウン / ドリルスルー

## 概要

集計テーブルの金額セルをクリックすると、その金額を構成する「明細科目 × 部署」の内訳をモーダル等で表示する機能。ドリルダウン（+展開で集計科目→明細科目を見る）は既に実装済み。ドリルスルーは別粒度（科目×部署）へのドリルである点が異なる。

## 要件（PRD §7.7 抜粋）

> 金額セルをクリックすると、明細科目 × 部署まで見える

- トリガー: 金額セル（`FinancialCell` / `DeltaCell`）のクリック
- 表示内容: 該当金額を構成する明細科目と部署のクロス集計
- UI形式: モーダル、ポップオーバー、サイドパネル等（未定、要検討）

## 現状

| 項目 | 状態 | 備考 |
|------|------|------|
| ドリルダウン（+展開） | ✅ 実装済み | `AggregateGroup` でアコーディオン展開 |
| ドリルスルー（金額クリック） | ❌ 未実装 | 本バックログの対象 |
| 金額セルのクリックハンドラ | ❌ なし | `FinancialCell` / `DeltaCell` は `<span>` のみ |
| 内訳表示UI | ❌ なし | モーダル・ポップオーバー等未実装 |
| 科目×部署クロス集計データ | ❌ 未定義 | `detailRows` は科目→行のマッピングのみ |

## 実装に必要な変更（想定）

### 1. データ層
- 科目 × 部署のクロス集計データ構造の定義
- 既存の `SummaryRow` または `ActualRow` からクロス集計を生成するロジック
- ※ 現在 `detailRows: Map<string, SummaryRow[]>` は「集計科目キー → 明細科目行の配列」のみ

### 2. UI層
- `FinancialCell` / `DeltaCell` にクリックハンドラを付与
- クリック可能であることを示すスタイル（cursor: pointer, hover効果等）
- 内訳を表示するコンポーネント（モーダル等）

### 3. 状態管理
- ドリルスルーで選択中のセル情報を保持するstate
- 表示/非表示のトグル制御
