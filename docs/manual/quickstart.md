# FPA Dashboard クイックスタートガイド

## 対象読者

経営層、事業部担当者、FPA メンバーなど、ダッシュボードを初めて開く方。

## このダッシュボードでできること

ログラスの月次データ（xlsx）をアップロードし、予実分析を画面上で行えます。ローカル開発環境ではテストデータを表示します。

- 全社と各事業部の予実比較を同一フォーマットで確認する
- 着地見込、YTD、単月の 3 つの時間軸を切り替えて分析する
- 主要科目の推移をグラフで把握する
- 差異の要因を分解して確認する
- 詳細表で集計科目から明細科目まで掘り下げる
- 管理画面で xlsx ファイルのアップロードとマスタ編集を行う

## 画面の見方

### 全体構成

画面左にサイドバー、中央にメインコンテンツが並びます。

```
┌────────┬──────────────────────────────────────────────┐
│サイドバー│              メインコンテンツ                  │
│        │  [全社|SaaS事業部|広告事業部|...]              │
│ 予実分析│  [着地見込|YTD|単月]                          │
│  管理  │                                            │
│        │  主要科目サマリー                              │
│        │  推移グラフ       差異分解グラフ                 │
│        │  詳細表                                      │
└────────┴──────────────────────────────────────────────┘
```

### サイドバー

| 項目 | 役割 |
|------|------|
| 予実分析 | 分析画面を開く。通常はここを使います |
| 管理 | データアップロード、マスタ編集、取込履歴の確認を行う（FPA 向け） |

### 組織タブ（画面上部）

全社、SaaS 事業部、広告事業部、EC 事業部ほか、横スクロールで事業部タブが並びます。タブをクリックすると、その組織のデータに切り替わります。

### 時間軸ピル（組織タブの下）

| ピル | 内容 |
|------|------|
| 着地見込 | 年度末までの見込と前月見込・前年実績の比較 |
| YTD | 対象月までの累計比較 |
| 単月 | 対象月単体の比較 |

初期表示は「着地見込」です。

### メインコンテンツ

1. **主要科目サマリー**: 売上高、売上原価、売上総利益、販管費、営業利益の 5 科目を一行ずつ表示。科目名をクリックすると推移グラフが連動します
2. **推移グラフ**: 選択した科目の月次推移を実績・見込の 2 系列で表示。対象月は縦の帯で強調されます。右上のトグルで金額と GMV 比率を切り替えられます
3. **差異分解グラフ**: 横棒グラフで差異要因を上位から表示。全社タブでは事業部ごと、事業部タブでは明細科目ごとに分解します。棒をクリックすると詳細表の該当行がハイライトされます
4. **詳細表**: 集計科目を起点に、`+` を押して明細科目を展開。列は A（前月見込）/ B（実績+見込）/ B-A / C（前年）/ B-C の文法に沿います

## 管理画面の基本操作

サイドバーの「管理」を開くと、データのアップロードとマスタ編集が行えます。

1. **データアップロード**: xlsx ファイルをドラッグ＆ドロップまたはファイル選択で指定します。シナリオと対象月はファイル内容から自動検出され、同一ラベルの既存データがある場合のみ上書き確認を経てアップロードを実行します
2. **マスタ編集**: 勘定科目マスタと部署マスタをタブで切り替えて編集します。各マスタでは明細科目名と集計科目名の対応、ソート順、bucket ステータス（通常 / 除外 / 未割当）を管理します
3. **取込履歴**: 過去のアップロード履歴を確認できます

## 基本操作（分析画面）

1. サイドバーの「予実分析」を開く
2. 組織タブで「全社」または対象の事業部を選ぶ
3. 時間軸ピルで「着地見込」「YTD」「単月」を切り替える
4. 主要科目サマリーで科目を選ぶと、推移グラフがその科目に切り替わる
5. 差異分解グラフの棒をクリックすると、詳細表の該当行が数秒間ハイライトされる
6. 詳細表の `+` を押すと、集計科目が明細科目まで展開される

## 注意点

- **GAS 環境とローカル環境でデータソースが異なります**: GAS 環境ではアップロードされた実データを表示します。ローカル開発環境ではフィクスチャデータ（テスト用固定データ）を表示します
- **デスクトップ環境を前提としています**: タブレットやスマートフォンでは正しく表示されない場合があります
