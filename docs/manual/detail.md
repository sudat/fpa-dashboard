# FPA Dashboard 詳細マニュアル

## 対象読者

FPA メンバー、運用担当者。ダッシュボードの全体像、前提、制約を把握したい方。

## 前提知識

- 予実分析業務の基本フローを理解していること
- ログラスから月次データ（xlsx）を取得する運用を知っていること
- 科目体系（集計科目・明細科目）と部署体系の概要を把握していること

## 前提環境

| 項目 | 内容 |
|------|------|
| 配信方式 | Google Apps Script（GAS）の HTML Service 経由で配信。フロントエンドは React / Vite でビルドし、clasp で GAS プロジェクトに push する |
| ブラウザ | モダンブラウザ（Chrome、Edge を推奨）。デスクトップ環境前提 |
| データ形式 | ログラスが出力する xlsx 形式（約 6 万行、**10 列固定**）。列: シナリオ, 年月度, 科目コード, 外部科目コード, 科目, 科目タイプ, 部署コード, 外部部署コード, 部署, 金額 |
| 認証 | GAS の標準的なアクセス制御を利用。画面内での独自認証機能は持たない |

## 画面構成

### 全体レイアウト

```
┌─────────────┬───────────────────────────────────────────┬─────────────┐
│  サイドバー   │              メインコンテンツ               │  コメント欄   │
│  (固定幅)    │                                           │  (準備中)    │
│             │  組織タブ: 全社 | SaaS事業部 | ...           │             │
│  予実分析     │  時間軸ピル: 着地見込 | YTD | 単月          │             │
│  管理       │                                           │             │
│             │  ① 主要科目サマリー                         │             │
│             │  ② 推移グラフ       ③ 差異分解グラフ         │             │
│             │  ④ 詳細表                                 │             │
└─────────────┴───────────────────────────────────────────┴─────────────┘
```

- サイドバーは薄く固定。トップレベルのナビゲーションは「予実分析」「管理」の 2 つのみ
- コメント欄は画面右側に固定幅で存在するが、現バージョンでは機能しない（後述）

### ナビゲーション

| メニュー | 遷移先 |
|---------|--------|
| 予実分析 | 分析画面（全社の着地見込が初期表示） |
| 管理 | 管理画面（データアップロード、マスタ編集、取込履歴） |

## 分析画面

### 組織タブ

画面最上部に、以下のタブが横スクロールで並ぶ。

| タブ | 内容 |
|------|------|
| 全社 | 全社合計の数値。トーン把握に使う。差異分解は事業部ごとに表示 |
| SaaS 事業部 | SaaS 事業部の数値。差異分解は明細科目ごとに表示 |
| 広告事業部 | 同上 |
| EC 事業部 | 同上 |
| フィンテック事業部 | 同上 |
| メディア事業部 | 同上 |
| インフラ事業部 | 同上 |
| データ事業部 | 同上 |
| グローバル事業部 | 同上 |
| 新規事業部 | 同上 |

全社と各事業部で同じ画面フォーマットを使用する。タブ切替で表示が切り替わるだけで、画面構造は変わらない。

### 時間軸ピル

組織タブのすぐ下に、ピル型の切替ボタンが 3 つ並ぶ。

| ピル | 計算内容 | 初期表示 |
|------|---------|---------|
| 着地見込 | A: 前月設定の年度末見込 / B: 実績+見込 / B-A / C: 前年実績 / B-C | 〇 |
| YTD | A: 前月設定の累計見込 / B: 実績累計 / B-A / C: 前年累計 / B-C | |
| 単月 | A: 前月設定の対象月見込 / B: 対象月実績 / B-A / C: 前年同月 / B-C | |

初期表示は「着地見込」。組織タブを切り替えても時間軸は維持される。逆も同様。

### 主要科目サマリー

画面上部に固定の主要科目を一行ずつ表示する。

| 科目 | タイプ |
|------|--------|
| 売上高 | 収益 |
| 売上原価 | 費用 |
| 売上総利益 | 収益 |
| 販管費 | 費用 |
| 営業利益 | 収益 |

表示列は以下の構成。

| 列 | 内容 |
|----|------|
| 科目名 | 集計科目名 |
| B（実績+見込） | 当期値 |
| A（前月見込） | 比較基準値 |
| B-A | 差異金額 |
| C（前年） | 前年同程度の値 |
| B-C | 前年差 |

科目名をクリック（選択）すると、**推移グラフだけ**がその科目に連動する。詳細表やコメントには影響しない。

### 推移グラフ

主要科目サマリーの下、左側に配置。

| 要素 | 仕様 |
|------|------|
| 横軸 | 月次時系列 |
| 縦軸 | 金額（初期）または GMV 比率（トグル切替） |
| 系列 | 実績・見込の 2 本 |
| 対象月強調 | 縦のハイライト帯で示す |
| 金額/GMV 比率切替 | グラフ右上のトグルボタン |
| クリック挙動 | グラフ上の月をクリックしても画面遷移はしない |

主要科目サマリーで科目を選んでいない場合、推移グラフにはデータが表示されない。

### 差異分解グラフ

推移グラフの右側に配置。

| 要素 | 仕様 |
|------|------|
| グラフ型 | 横棒グラフ |
| 分解軸 | 全社タブ: 事業部ごと / 事業部タブ: 明細科目ごと |
| 表示本数 | 差異影響の大きい上位 5 本前後 |
| 並び順 | 差異絶対値の大きい順 |
| 色分け | 良化 / 悪化の 2 色 |
| クリック挙動 | 棒をクリックすると詳細表の該当行が数秒間ハイライトされる（弱連動） |

棒クリック時の弱連動について:
- 詳細表が別タブや別モーダルに切り替わることはない
- 該当集計科目が必要に応じて自動展開され、一時的にハイライトされる
- ハイライトは約 3 秒で自動的に消える
- コメントや他のパネルには影響しない

### 詳細表

画面下部に配置。

| 要素 | 仕様 |
|------|------|
| 行構成 | 集計科目を起点に、`+` 展開で明細科目を表示 |
| 列構成 | A / B / B-A / C / B-C（現行業務の文法を踏襲） |
| 時間軸 | 着地見込、YTD、単月で列の並びはほぼ共通 |
| ソート | 明細科目は科目名の昇順 |

## 管理画面

サイドバーの「管理」をクリックすると表示される。

管理画面は 3 つの領域で構成される。データアップロード、マスタ編集、取込履歴の確認が行える。

### データアップロード

xlsx ファイルのアップロードを受け付ける。

| 要素 | 仕様 |
|------|------|
| ファイル形式 | xlsx（.xls も可） |
| 入力方法 | ドラッグ＆ドロップまたはファイル選択ボタン |
| シナリオ入力 | シナリオ種別（実績 / 予算 / 見込）、対象月、見込開始月（見込の場合）を入力 |
| プレビュー | アップロード前にデータ件数、含まれる部署・科目の一覧を確認可能 |
| 上書き警告 | 同一シナリオ種別・ラベルの既存アップロードが存在する場合、上書き確認を表示 |
| 実行 | プレビュー確認後、「アップロード実行」でコミット。GAS バックエンドが xlsx をパースしてスプレッドシートに保存 |

アップロードの流れ:

1. xlsx ファイルを選択する
2. シナリオ種別と対象月を入力する
3. 「プレビュー」でデータ概要を確認する
4. 内容に問題がなければ「アップロード実行」を押す
5. 成功またはエラーのステータスが表示される

### マスタ編集

勘定科目マスタと部署マスタをタブで切り替えて編集する。

#### 勘定科目マスタ

明細科目名と集計科目名の対応関係を管理する。

| 編集項目 | 内容 |
|---------|------|
| 明細科目名 | ログラスの科目名（キー） |
| 集計科目名 | 対応する集計科目名（売上高、売上原価など） |
| ソート順 | 集計科目内での表示順 |
| GMV 対象 | その科目が GMV 集計の分母に含まれるか |
| bucket ステータス | 通常 / 除外 / 未割当 |

#### 部署マスタ

部署名と事業部名の対応関係を管理する。

| 編集項目 | 内容 |
|---------|------|
| 部署名 | ログラスの部署名（キー） |
| 事業部名 | 対応する事業部名 |
| ソート順 | 事業部内での表示順 |
| bucket ステータス | 通常 / 除外 / 未割当 |

#### 保存バー

マスタ編集画面の下部に固定表示される。

- 変更がある場合のみ「保存」ボタンが有効になる
- 未割当・除外のエントリ数を表示する
- 保存状態（保存済み / 保存中 / エラー）を表示する

### 取込履歴

データ取込の結果一覧を表示する。

| 列 | 内容 |
|----|------|
| アップロード ID | 一意の識別子 |
| タイムスタンプ | アップロード実行日時 |
| アップロード者 | 実行ユーザーのメールアドレス |
| シナリオ | シナリオ種別と生成ラベル |
| ファイル名 | アップロードされたファイル名 |

### 分析画面への導線

管理画面の右下に「分析画面で確認」ボタンがある。これをクリックすると予実分析画面に遷移する。FPA は取込結果とマスタ差分を確認した後、実際の数値を分析画面で検証する運用を想定している。

## データ形式

### xlsx の列構成（10 列固定）

ログラスから出力される xlsx ファイルは、以下の 10 列で構成される。

| 列番号 | 列名 | 内容 |
|--------|------|------|
| 1 | シナリオ | 計画・実績・見込などのシナリオ種別 |
| 2 | 年月度 | 対象年月（yyyy-MM 形式） |
| 3 | 科目コード | ログラスの科目コード |
| 4 | 外部科目コード | 外部システムの科目コード |
| 5 | 科目 | 科目名 |
| 6 | 科目タイプ | 収益 / 費用 / 資産 / 負債 / その他 |
| 7 | 部署コード | ログラスの部署コード |
| 8 | 外部部署コード | 外部システムの部署コード |
| 9 | 部署 | 部署名 |
| 10 | 金額 | 数値 |

### マスタによるデータ拡張

生の 10 列データは、マスタマッピングを通じて以下の情報が付与される。

- **集計科目名**: 明細科目名に対応する集計科目名（売上高、売上原価など）
- **ソート順**: 集計科目内の表示順
- **GMV 対象フラグ**: GMV 分母に含まれるか
- **bucket ステータス**: 通常（分析に含む）/ 除外（分析から除外）/ 未割当（マスタ未設定）
- **事業部名**: 部署名に対応する事業部名

bucket ステータスが「除外」の科目・部署は分析画面に表示されない。「未割当」のデータは一括して「未割当」として集計される。

## データ前提

### 現在の状態

分析画面のデータソースは、実行環境によって自動的に切り替わる。

| 環境 | データソース | 内容 |
|------|-------------|------|
| GAS 環境（本番） | GAS バックエンド | アップロードされた xlsx データを GAS の Spreadsheet Service 経由で取得。マスタマッピング済みのデータを分析に使用 |
| ローカル開発環境 | フィクスチャデータ | 固定のテストデータセットを表示。GAS API が利用できない場合に自動的にフォールバック |

GAS 環境では、3 つのシナリオファミリ（actual / budget / forecast）ごとにバックエンドからデータを取得し、マスタマッピングと bucket フィルタリングを適用した上で集計・比較処理を行う。

ローカル開発環境では、フィクスチャデータ（固定のテストデータセット）を使用する。分析対象月は `2026-02` に固定。

### データパイプライン

```
xlsx アップロード
      ↓
GAS バックエンドでパース（10 列の raw データに変換）
      ↓
スプレッドシート（ImportData シート）に保存
      ↓
分析画面でデータ取得（シナリオファミリごと）
      ↓
マスタマッピング適用（集計科目名、ソート順、bucket ステータスを付与）
      ↓
bucket フィルタリング（除外データを除去、未割当データを統合）
      ↓
正規化（scenarioKey, yearMonth, 科目, 部署を統一フォーマットに）
      ↓
集計（集計科目、売上総利益、営業利益を計算）
      ↓
比較データ生成（A/B/C 列の構築）
      ↓
画面表示
```

ローカル開発環境では、フィクスチャデータからこのパイプラインの「正規化」以降を実行する。

## 制約

| 項目 | 内容 |
|------|------|
| データソース | GAS 環境では実データ、ローカル開発環境ではフィクスチャデータを表示 |
| コメント機能 | 右サイドに領域は確保されているが、投稿・閲覧は不可 |
| 管理画面 | xlsx アップロード、マスタ編集、取込履歴の確認が可能 |
| スナップショット | データの版管理機能はない |
| 通知 | Slack 通知やメール通知は未実装 |
| モバイル対応 | デスクトップ前提のレイアウト。タブレット・スマートフォンでは表示が崩れる可能性がある |
| 対象月 | 2026-02 に固定。ユーザーによる対象月の切替は不可 |

## 用語

| 用語 | 意味 |
|------|------|
| 着地見込 | 年度末までの見込値。実績確定月までは実績、以降は見込 |
| YTD | Year-to-Date。年初からの累計 |
| 単月 | 対象月単体の値 |
| A | 前月に設定した見込値（比較基準） |
| B | 実績値または実績+見込値 |
| C | 前年同程度の値 |
| B-A | 当期と前月見込の差異 |
| B-C | 当期と前年の差異 |
| 集計科目 | 売上高、売上原価など、複数の明細科目を束ねた科目 |
| 明細科目 | 集計科目を構成する個別の科目 |
| 弱連動 | 差異分解グラフのクリックで詳細表の該当行を一時的にハイライトする挙動。画面遷移や状態の永続的な変更を伴わない |
| フィクスチャ | 開発・テスト用の固定データセット |
| GMV 比率 | 分母を GMV 集計科目とした比率表示 |
| bucket ステータス | 科目・部署の分析対象区分。通常（分析に含む）、除外（分析から除外）、未割当（マスタ未設定）の 3 種 |
| シナリオファミリ | actual（実績）、budget（予算）、forecast（見込）の 3 種のシナリオ区分 |

## 想定運用フロー

### 現在の運用

1. GAS 経由でダッシュボードにアクセスする
2. 管理画面で xlsx ファイルをアップロードする（シナリオ種別・対象月を指定）
3. 取込履歴でアップロード結果を確認する
4. マスタ編集で科目・部署のマッピングを調整する（新規科目や部署が追加された場合）
5. 分析画面で数値の整合性を検証する
6. 組織タブ、時間軸ピルを切り替えてデータを確認する
7. 主要科目を選んで推移グラフを確認する
8. 差異分解グラフで差異要因を確認し、詳細表を展開して明細を見る

### 今後の運用（追加予定機能）

1. 事業部がコメントを入力する（今後対応）
2. FPA が論点整理・精査・示唆追加を行う（今後対応）
3. 経営層が確認し、予実分析会に入る
4. 通知連携で関係者に変更を知らせる（今後対応）
5. 承認ワークフローで承認プロセスを管理する（今後対応）
