# FPA Dashboard 詳細マニュアル

## 対象読者

FPA メンバー、運用担当者。ダッシュボードの全体像、前提、制約を把握したい方。

## 前提知識

- 予実分析業務の基本フローを理解していること
- ログラスから月次データ（xlsx）を取得する運用を知っていること
- 科目体系（集計科目・明細科目）と部署体系の概要を把握していること

## 前提環境

| 項目 | 内容 |
|------|------|
| 配信方式 | Google Apps Script（GAS）の HTML Service 経由で配信。フロントエンドは React / Vite でビルドし、clasp で GAS プロジェクトに push する |
| ブラウザ | モダンブラウザ（Chrome、Edge を推奨）。デスクトップ環境前提 |
| データ形式 | ログラスが出力する xlsx 形式（約 6 万行、10 列前提） |
| 認証 | GAS の標準的なアクセス制御を利用。画面内での独自認証機能は持たない |

## 画面構成

### 全体レイアウト

```
┌─────────────┬───────────────────────────────────────────┬─────────────┐
│  サイドバー   │              メインコンテンツ               │  コメント欄   │
│  (固定幅)    │                                           │  (準備中)    │
│             │  組織タブ: 全社 | SaaS事業部 | ...           │             │
│  予実分析     │  時間軸ピル: 着地見込 | YTD | 単月          │             │
│  管理       │                                           │             │
│             │  ① 主要科目サマリー                         │             │
│             │  ② 推移グラフ       ③ 差異分解グラフ         │             │
│             │  ④ 詳細表                                 │             │
└─────────────┴───────────────────────────────────────────┴─────────────┘
```

- サイドバーは薄く固定。トップレベルのナビゲーションは「予実分析」「管理」の 2 つのみ
- コメント欄は画面右側に固定幅で存在するが、現バージョンでは機能しない（後述）

### ナビゲーション

| メニュー | 遷移先 |
|---------|--------|
| 予実分析 | 分析画面（全社の着地見込が初期表示） |
| 管理 | 管理画面（取込ログ＋マスタ差分警告） |

## 分析画面

### 組織タブ

画面最上部に、以下のタブが横スクロールで並ぶ。

| タブ | 内容 |
|------|------|
| 全社 | 全社合計の数値。トーン把握に使う。差異分解は事業部ごとに表示 |
| SaaS 事業部 | SaaS 事業部の数値。差異分解は明細科目ごとに表示 |
| 広告事業部 | 同上 |
| EC 事業部 | 同上 |
| フィンテック事業部 | 同上 |
| メディア事業部 | 同上 |
| インフラ事業部 | 同上 |
| データ事業部 | 同上 |
| グローバル事業部 | 同上 |
| 新規事業部 | 同上 |

全社と各事業部で同じ画面フォーマットを使用する。タブ切替で表示が切り替わるだけで、画面構造は変わらない。

### 時間軸ピル

組織タブのすぐ下に、ピル型の切替ボタンが 3 つ並ぶ。

| ピル | 計算内容 | 初期表示 |
|------|---------|---------|
| 着地見込 | A: 前月設定の年度末見込 / B: 実績+見込 / B-A / C: 前年実績 / B-C | 〇 |
| YTD | A: 前月設定の累計見込 / B: 実績累計 / B-A / C: 前年累計 / B-C | |
| 単月 | A: 前月設定の対象月見込 / B: 対象月実績 / B-A / C: 前年同月 / B-C | |

初期表示は「着地見込」。組織タブを切り替えても時間軸は維持される。逆も同様。

### 主要科目サマリー

画面上部に固定の主要科目を一行ずつ表示する。

| 科目 | タイプ |
|------|--------|
| 売上高 | 収益 |
| 売上原価 | 費用 |
| 売上総利益 | 収益 |
| 販管費 | 費用 |
| 営業利益 | 収益 |

表示列は以下の構成。

| 列 | 内容 |
|----|------|
| 科目名 | 集計科目名 |
| B（実績+見込） | 当期値 |
| A（前月見込） | 比較基準値 |
| B-A | 差異金額 |
| C（前年） | 前年同程度の値 |
| B-C | 前年差 |

科目名をクリック（選択）すると、**推移グラフだけ**がその科目に連動する。詳細表やコメントには影響しない。

### 推移グラフ

主要科目サマリーの下、左側に配置。

| 要素 | 仕様 |
|------|------|
| 横軸 | 月次時系列 |
| 縦軸 | 金額（初期）または GMV 比率（トグル切替） |
| 系列 | 実績・見込の 2 本 |
| 対象月強調 | 縦のハイライト帯で示す |
| 金額/GMV 比率切替 | グラフ右上のトグルボタン |
| クリック挙動 | グラフ上の月をクリックしても画面遷移はしない |

主要科目サマリーで科目を選んでいない場合、推移グラフにはデータが表示されない。

### 差異分解グラフ

推移グラフの右側に配置。

| 要素 | 仕様 |
|------|------|
| グラフ型 | 横棒グラフ |
| 分解軸 | 全社タブ: 事業部ごと / 事業部タブ: 明細科目ごと |
| 表示本数 | 差異影響の大きい上位 5 本前後 |
| 並び順 | 差異絶対値の大きい順 |
| 色分け | 良化 / 悪化の 2 色 |
| クリック挙動 | 棒をクリックすると詳細表の該当行が数秒間ハイライトされる（弱連動） |

棒クリック時の弱連動について:
- 詳細表が別タブや別モーダルに切り替わることはない
- 該当集計科目が必要に応じて自動展開され、一時的にハイライトされる
- ハイライトは約 3 秒で自動的に消える
- コメントや他のパネルには影響しない

### 詳細表

画面下部に配置。

| 要素 | 仕様 |
|------|------|
| 行構成 | 集計科目を起点に、`+` 展開で明細科目を表示 |
| 列構成 | A / B / B-A / C / B-C（現行業務の文法を踏襲） |
| 時間軸 | 着地見込、YTD、単月で列の並びはほぼ共通 |
| ソート | 明細科目は科目名の昇順 |

## 管理画面

サイドバーの「管理」をクリックすると表示される。

管理画面は最小構成であり、以下の 2 つの領域を表示する。いずれも現時点ではモックデータを表示しており、実際のアップロードや取込処理は行えない。

### 取込ログ

データ取込の結果一覧を表示する（現時点ではモックデータ）。

| 列 | 内容 |
|----|------|
| ファイル名 | 取込対象のファイル名 |
| 取込日時 | アップロード実行日時 |
| 件数 | 取込行数 |
| ステータス | 成功 / 部分成功 / 失敗 |

部分成功や失敗の場合はエラーメッセージが併記される。

### マスタ差分警告

データ取込時に検知される、科目・部署の差分を表示する（現時点ではモックデータ）。

| 列 | 内容 |
|----|------|
| 種別 | 新規科目 / 新規部署 / マッピング変更 |
| コード | 科目コードまたは部署コード |
| 名称 | 科目名または部署名 |
| ステータス | 未処理 / 承認済 / 除外 |

### 分析画面への導線

管理画面の右下に「分析画面で確認」ボタンがある。これをクリックすると予実分析画面に遷移する。FPA は取込結果とマスタ差分を確認した後、実際の数値を分析画面で検証する運用を想定している。

## データ前提

### 現在の状態

分析画面に表示されているデータは、**フィクスチャデータ（テスト用固定データ）** です。ログラスからの実際の xlsx アップロード連携は未実装です。

- `loglassSmallRawFixture` という固定データセットをソースとしている
- 分析対象月は `2026-02` に固定
- 事業部コードは `D001`, `D002` ... の形式で内部的に採番
- 主要科目（売上高、売上原価、売上総利益、販管費、営業利益）は、フィクスチャ内の明細科目から集計して算出している

### 将来のデータフロー（想定）

以下は、本番運用時に想定されるフローである。現時点では実装されていない部分が多い。

1. ログラスから xlsx ファイルを取得する
2. 管理画面でアップロードする（**未実装**）
3. 正規化処理で取込・集計する（**ロジックは実装済み、アップロード経由の呼出は未接続**）
4. マスタ差分を警告表示する（**UI 枠組みはあり、モックデータを表示**）
5. FPA が取込ログと警告を確認する（**UI 枠組みはあり、モックデータを表示**）
6. 分析画面で数値を検証する（**フィクスチャデータによる表示は可能**）
7. 事業部へ公開する（**未実装**）

現在実装されているのは、ステップ 3 の正規化ロジック（コード上に存在）と、ステップ 6 の分析画面 UI（フィクスチャデータを表示）に限る。アップロード機能（ステップ 2）、本番データの取込経路、公開フロー（ステップ 7）は未実装である。

## 制約

| 項目 | 内容 |
|------|------|
| データソース | フィクスチャデータのみ。実際のアップロード API には接続していない |
| コメント機能 | 右サイドに領域は確保されているが、投稿・閲覧は不可 |
| 管理画面のアップロード操作 | 取込ログとマスタ差分警告の表示のみ。アップロード UI はない |
| スナップショット | データの版管理機能はない |
| 通知 | Slack 通知やメール通知は未実装 |
| モバイル対応 | デスクトップ前提のレイアウト。タブレット・スマートフォンでは表示が崩れる可能性がある |
| 対象月 | 2026-02 に固定。ユーザーによる対象月の切替は不可 |

## 用語

| 用語 | 意味 |
|------|------|
| 着地見込 | 年度末までの見込値。実績確定月までは実績、以降は見込 |
| YTD | Year-to-Date。年初からの累計 |
| 単月 | 対象月単体の値 |
| A | 前月に設定した見込値（比較基準） |
| B | 実績値または実績+見込値 |
| C | 前年同程度の値 |
| B-A | 当期と前月見込の差異 |
| B-C | 当期と前年の差異 |
| 集計科目 | 売上高、売上原価など、複数の明細科目を束ねた科目 |
| 明細科目 | 集計科目を構成する個別の科目 |
| 弱連動 | 差異分解グラフのクリックで詳細表の該当行を一時的にハイライトする挙動。画面遷移や状態の永続的な変更を伴わない |
| フィクスチャ | 開発・テスト用の固定データセット |
| GMV 比率 | 分母を GMV 集計科目とした比率表示 |

## 想定運用フロー

### 現在の運用（フィクスチャ前提）

1. GAS 経由でダッシュボードにアクセスする
2. 分析画面が開き、全社の着地見込が表示される
3. 組織タブ、時間軸ピルを切り替えてデータを確認する
4. 主要科目を選んで推移グラフを確認する
5. 差異分解グラフで差異要因を確認し、詳細表を展開して明細を見る

### 将来の運用（データ取込実装後の想定）

1. 管理画面で xlsx ファイルをアップロードする
2. 取込ログで結果を確認する
3. マスタ差分警告で新規科目・新規部署を確認する
4. 「分析画面で確認」ボタンで分析画面に遷移する
5. 数値の整合性を分析画面で検証する
6. 事業部へ公開する
7. 事業部がコメントを入力する（今後対応）
8. FPA が論点整理・精査・示唆追加を行う（今後対応）
9. 経営層が確認し、予実分析会に入る
